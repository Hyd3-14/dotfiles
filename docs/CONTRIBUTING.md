# Contributing / 運用ガイド

このリポジトリは個人用 dotfiles です。外部からの貢献は想定していませんが、自分（または将来の自分）が安全に・わかりやすく変更できるように運用ルールとインストール手順をここにまとめます。

## ブランチ運用（個人向けの推奨）

- シンプル運用：main ブランチのみでも運用可能です（小さな修正が中心のとき）。
- 安全重視（推奨）：main と短命の feature ブランチを併用する運用が最も扱いやすいです。
  - 小さな変更は main に直接コミットしても OK。
  - 大きな変更（設定の大幅な差し替え、スクリプトのリファクタなど）は `feature/<説明>` ブランチを作って作業する。
  - テスト完了後、main にマージしてブランチを削除する。
- タグ：重要な状態（例：大きく動いたときのスナップショット）はタグを打っておくと復旧が容易になります。
  - 例: `git tag v2026-01-03` → push: `git push origin v2026-01-03`

## install.sh（使い方）

リポジトリに含まれる `scripts/install.sh` は、以下を自動で行います（デフォルトの動作）：

- 主要な設定ファイルをホームディレクトリにシンボリックリンクで配置
- 既存の設定ファイルがある場合はバックアップ（タイムスタンプ付きディレクトリ）
- `scripts/bin/` 配下で実行権のあるファイルを `~/.local/bin`（既定）にシンボリックリンク

重要：スクリプトはファイルの移動や上書きを伴うため、実行前に必ず内容を確認してください。

### 主要オプション

- `--dry-run` または `-n`
  実際には変更せず、何を行うかだけ表示します。最初は必ずこれで確認してください。
- `--force` または `-f`
  既存の symlink を置き換えるなど、強制的に差し替えます。慎重に使ってください。
- `--yes` または `-y`
  対話プロンプトに自動的に「yes」で応答します（非対話実行に便利）。
- `--help` または `-h`
  ヘルプ表示。

### 実行例

- ドライラン（変更内容の確認）
  ./scripts/install.sh --dry-run
- 実行（対話あり）
  ./scripts/install.sh
- 実行（対話なし）
  ./scripts/install.sh --yes

### 補助スクリプト

- `scripts/bootstrap-ubuntu.sh`
  - Ubuntu/WSL の依存コマンド確認と apt インストール補助
- `scripts/import-home-config.sh`
  - 現在のホーム設定から、管理対象の設定を dotfiles に取り込む

### バックアップ

- 既存ファイルは `~/dotfiles_backup_<timestamp>/` に移動します。
- スクリプト実行後は必ずバックアップ先を確認してください。

## install.sh の拡張（ファイルの追加方法）

- 現状、`scripts/install.sh` は `scripts/links.sh` のマッピング定義を読み込んでリンクします。
- 新しい設定ファイルを追加したら、`scripts/links.sh` の `config_files` または `add_link` に追記してください。例：
  ```bash
  add_link "$repo_root/tmux/.tmux.conf" "$HOME_DIR/.tmux.conf"
  ```
- 追加後は `--dry-run` で動作を確認してから実行してください。

## 安全に使うためのチェックリスト

1. スクリプトを読む（特に mv, ln, rm を行う箇所）。
2. `--dry-run` で予定動作を確認する。
3. バックアップを手元にコピーしてから（必要なら）実行する。
4. 実行後に `~/.local/bin` が PATH に入っているか確認する（例: `export PATH="$HOME/.local/bin:$PATH"`）。
